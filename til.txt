git 2주차 내용 
1. 이슈 issue는 누가 프로젝트의 어떤 부분을 할지 정하는 것 
#? 이런식으로 되어있으며 커밋할때 입력해주면 이슈에 그 커밋이 보여서 타인의 의견반영도 가능

2. 브랜치 branch는 하나하나가 작업 환경이며 나뭇가지처럼 여러개로 뻗어나갈수 있다.
나중에 다른 브랜치랑 병합도 가능하며 보통 main 브랜치 하나를 두고 작업 브랜치 여러개를 써서 
main 브랜치에 병합하는 경우가 많을것 같다.

3.머지 merge는 위에 언급된 병합하는 기능이며 병합할때 브랜치에 있던 커밋 도움말도 같이 딸려온다.

4. 컨플릭트 conflict 는 위에 머지할때 완전히 같은 이름의 파일이 겹치면 발생하는데 
발생하면 				ex) sub1 브랜치의 hello world.py을 main 브랜치의 hello world.py 파일에 머지해줬을때 
============head		==========================          

머지 받는 브랜치의 내용       		print('hello world!')         
================                  ==========================

머지해서 주는 브랜치의 내용   	a = 'hello world!'                
                                     		print(a)            
=======주는 브랜치 이름 		========sub1

이런식으로 나오는데 

====이 포함된 것을 다 지워주면 정상적으로 작동한다. 

5. 로컬 저장소는 원격 저장소를 tracking 하고 있으며 이로 인해 push와 pull 기능이 사용 가능한것이다. 

5-1. 원격저장소는 로컬 저장소에서 push 받을때 원격 저장소에 해당 브랜치와 동일한 이름의 브랜치가 없으면 
자동으로 해당 이름의 브랜치를 만들어 거기에 저장해준다.
