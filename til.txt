git 1주차 배운것 

1. git이란 버전관리 도구겸 로컬 저장소 

2. sourcetree는 git을 편하게 사용하기위한 도구 

3. git초기화(프로젝트 생성)는 sourcetree에서 create버튼 클릭 후 경로 및 이름 입력 후 git 형식으로 프로젝트 생성하는것 

4. commit(버전관리) 은 언제 누가 어떻게 변경되었는지 커밋 메모를 적은 후 그 변경된 파일을 스테이징해서 기록하는것 

4-1. staging 은 수정된 파일을 git에 올려서 저장하는것 (2개 수정됐는데 1개만 스테이징도 가능) 

5. github는 오픈소스이며 타인과 같이 작업하기위해 로컬 저장소에서 커밋을 주고 받고(pull / push) 하는 원격 저장소 

5-1. push는 로컬 저장소에서 원격 저장소에 커밋을 보내주고 반영시켜주는것

5-2 .pull은 원격 저장소에서 로컬 저장소에 커밋을 보내주고 반영시키는것      보통 원격저장소에서 변경사항이 생기면 pull쪽에 알림이 생김 

6. clone은 기존프로젝트에 새 인원이 참가하여 기존 프로젝트의 내용을 확인할때 
기존 원격 저장소와 새 로컬 저장소를 연결 시키고 원격 저장소에 커밋들을 로컬저장소에 보내주고 반영시켜주는것
